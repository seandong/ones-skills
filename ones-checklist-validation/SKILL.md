# ONES Checklist Validation（同事可直接执行版）

## 验证范围（固定）
- ✅ 2.1 登录/登出
- ✅ 2.2 项目管理
- ✅ 2.3 知识库管理
- ❌ 不执行：2.4 邮箱配置、2.5 多域名、3.x 标品插件

## 红线（必须遵守）
- 禁止启停插件、改权限、改系统配置。
- 默认只读核验；涉及新建/上传/发布等写操作，需先确认授权。
- 报告必须输出：**通过 / 失败 / 阻塞**，并附证据路径。

## 执行前准备
- ONES 地址可访问（示例：`http://<ip>:30011`）
- 管理员账号可登录
- 本机可运行 Node.js + Chrome/Chromium
- 工作目录：`/tmp/ones-check`

### 推荐运行模式（避免悬浮菜单/下拉菜单不稳定）
优先使用 **headed + Xvfb**（而不是纯 headless）：

```bash
apt-get update
DEBIAN_FRONTEND=noninteractive apt-get install -y \
  xvfb xauth fonts-noto-cjk libnss3 libgtk-3-0 libxss1 libasound2
```

执行自动化时建议：

```bash
xvfb-run -a -s '-screen 0 1920x1080x24' node <your-script>.js
```

浏览器启动参数建议：
- `headless: false`
- `args: ['--no-sandbox','--disable-dev-shm-usage']`

## 快速执行建议（自动化）
- 先跑基础核验（2.1+2.2）
- 再跑创建流程收口探针（解决“创建成功但未确认”）
- 最后补 2.3 知识库核验（含“新增页面并保存”）
- 知识库写操作优先走“编辑态发布”路径：先点“编辑”进入专门编辑页，再填标题/正文后点“发布”
- 统一汇总到一份最终报告

## 常见阻碍与处理（发同事必看）
1. **卡在项目创建页，未真正创建成功**
   - 先看是否有“项目标识不能为空/不合法”。
   - 补齐合法标识后再点下一步/新建项目。

2. **知识库“➕”菜单点不开（悬浮菜单问题）**
   - 先确认使用了 `xvfb-run + headless:false`。
   - 优先点击目录区右上角 `ones-dropdown-trigger` 元素；必要时采用键盘 `ArrowDown + Enter` 选择第一项。
   - 仍失败则标记“阻塞（UI 悬浮菜单自动化不稳定）”，改半人工（人工点开，自动化继续取证）。

3. **找不到“系统安全/操作日志”独立菜单**
   - 可能是版本/权限差异；只要能进入团队设置域并完成只读核验，可判“通过（菜单结构差异）”。

4. **发布后无明确成功文案**
   - 允许用“按钮状态变化 / 页面状态变化 / URL变化”作为辅助证据。

5. **环境依赖问题**
   - 缺字体或图形库会导致元素渲染异常，先安装依赖再执行。
   - 若系统提示内核升级（如 `5.15.0-171`），不影响当前验证，但建议维护窗口重启。

6. **知识库“发布”点击后仍停留未发布/草稿态**
   - 现象：点击“发布”后页面仍显示“未发布”，URL 仍为 `/draft/...`。
   - 常见原因：不在真正“可发布编辑态”，或标题/正文未形成有效可发布内容。
   - 处理步骤（按顺序）：
     1) 先点“编辑”进入专门编辑页（应看到标题输入 + 正文输入 + 顶部发布）。
     2) 填写标题（非占位符）并在正文输入至少一段文本。
     3) 再点“发布”；若无变化，刷新后重复“编辑 → 补内容 → 发布”。
   - 通过判定（命中任一组即可）：
     - 操作区从“发布/未发布”切换为“编辑/共享”；
     - 左侧目录出现新页面标题；
     - URL 从 `/draft/...` 切换为 `/page/...`（或等效已发布页面路由）。
